## 구현할 기능 정리

### 입출력
- [x] 메뉴 추천을 받을 코치의 이름을 입력
- [ ] 각 코치가 못 먹는 메뉴를 입력
- [ ] 서비스 시작 문구 출력
- [ ] 서비스 종료 문구(결과) 출력
- [ ] 예외 상황 시 에러 문구 출력

### 검증
- [ ] 코치의 이름이 2글자이상 4글자 이하인치 검증
- [ ] 코치가 2명이상 5명 이하인지 검증
- [ ] 코치가 못먹는 음식이 2개 이하인지 검증
- [ ] 코치가 못먹는 음식이 미션에서 제공된 메뉴판에 있는지 검증
- [선택] 각 코치의 메뉴가 같은카테고리가 2회 이하인지, 중복되는 메뉴가 없는지 검증 // 이미 재추점 로직이 있고 사용자 입력이 아니므로 필수 아님 

### 카테고리
- [x] 주어진 카테고리 목록을 저장해놓는 저장소 구현
- [x] 요일별 카테고리 저장소는 랜덤으로 생성된다.

### 코치
- [x] 각 코치는 요일별 카테고리에 맞는 메뉴와 못먹는 메뉴를 가지고 있다.
- [x] 중복되는 메뉴와 못먹는 메뉴는 재추첨한다.


### 요일별 카테고리
- [x] 매 요일마다 하나의 카테고리를 선정한다.
- [x] 같은 카테고리가 3번째 나올 경우 랜덤으로 재생성한다.
